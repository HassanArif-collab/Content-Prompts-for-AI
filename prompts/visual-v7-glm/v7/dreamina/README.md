# Dreamina reference (archived — off for now)

Dreamina no longer gives free credits, so it is **not used** in the current workflow.
It is kept here on purpose — the hard-won know-how (exact UI flow, GPT Image 2 gotchas,
credit gates) is worth preserving for when Dreamina (or its credits) come back.

**Active generation instead:** Flow (Nano Banana 2 for stills, frames/ingredients for
motion) + ChatGPT image, generated by the user and dropped into the visuals folder, which
the app auto-links. See the "Image & animation workflow" and "Motion technique" sections in
`../Visuals Generation Prompt v7.md`.

## Where the Dreamina guide lives (nothing was deleted)
- `../agent_asset_generation.md` — "Browser Task Contract", the Dreamina UI cautions
  (AI Image vs AI Agent, GPT Image 2 selection + 16:9 order, 4-images-per-run batch,
  credit gates), and "§2 Generated Still Images (Dreamina)".
- `../browser_capabilities.md` — Dreamina capability definitions.
- `../browser_runtime.md` — Scout Mode / logged-in UI runtime notes.
- `../../tools/dreamina/dreamina_download_latest.js`, `dreamina_image_probe.js` — the
  automation scripts.

## To re-enable Dreamina later
1. Remove the "Dreamina is OFF" status notes in `../Visuals Generation Prompt v7.md`
   and the banner in `../agent_asset_generation.md`.
2. Re-allow `dreamina.*` capabilities in the asset workflow.
3. The `POST /api/ai/browse/run_task` route + `tools/dreamina/` scripts are still present.
